【GatsbyJS入門】
GatsbyJSのインストール

概要

GatsbyJSとは？

  • 高速表示が期待できる静的サイトジェネレータのこと。

GatsbyJSの利点と課題

  • 利点
    • 表示が高速。静的サイトゆえに、一般的なこれまでの CMSのように直接的に DBを使用してサーバ上で HTMLファイルの生成をしない。また、Reactをベースにしているので、ページ内の部分的な更新処理ができるため速い。
    • 動的な表示もできる。基本的に静的なサイトだが、Headless CMSや GraphQLなどを利用することで動的な表示も可能。
    • 習熟度にあわせて導入できる。すでに用意されている Starterやテーマを利用すれば、エンジニアでなくても難しくはない。ゼロから制作する場合は、エンジニアであれば難しくないが、やりたいことの分だけ難易度は増していく。
  • 課題
    • 更新頻度が高いサイトや大規模なサイトには向かない。Next.jsなどを利用するほうがいい。棲み分け。
    • 広く利用されてはいないので、導入に不安がある。情報量、継続性などは現状においてまだまだ懸念される。

GatsbyJSのインストール手順（Windows版）

コマンドラインとNode.js

  • 事前に、コマンドラインツールとNode.jsをインストールしておく。

GatsbyJS

  1. 管理者権限で Git Bashを起動する。sudoなどが使える場合は、標準起動でも問題ない。臨機応変に対応されたし。
  2. GatsbyJSの CLIツールをインストールする。CLI（コマンドラインインタフェース）をインストールすると、GatsbyJS独自のコマンドを使うことができるようになる。
  3. Starterを利用してプロジェクトを作成する。GatsbyJSの作業場所（ディレクトリ名）を指定し、Starterは、Gatsby Starter Libraryで案件にあったものを探すといい。
  4. 利用する Starterの内容によっては、ダウンロードに時間がかかるかもしれない。
  5. ダウンロードが完了したら作成したディレクトリへ移動し、GatsbyJS開発環境を起ち上げる。
  6. ブラウザに、http://localhost:8000/ を入力して表示する。今回の場合は、「Hello World!」と表示されれば成功。
  7. Gitの対象から除外したい場合は、.gitignoreを修正。

// GatsbyJSの CLIツールをインストール。
$ npm install -g gatsby-cli

// Starterを使用したプロジェクトの作成。
// gatsby new [ディレクトリ名] [StarterのURL]
$ gatsby new gatsby-test https://github.com/gatsbyjs/gatsby-starter-hello-world

// GatsbyJS開発環境の起ち上げ。
$ cd gatsby-test
$ gatsby develop
