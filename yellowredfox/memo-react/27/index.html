【React Hooks】
useState()とは？

デモ

コード概要

要点

  • 以前 Class Componentで作成したカウントアップボタンを、Function Componentで書き直してみた。Class Componentではいわゆる「State」を使用するわけだが、Function Componentでは代わりに、React Hooksの useState()を使うことで解決する。
  • Reactのコードを inde.jsxなどの外部ファイルに書き出しており、かつ React Hooksを使用する場合は、importや exportで指定する必要がある。今回は HTMLファイルに直接書いているのでその必要はない。
  • useStateの定義は「const [ 状態変数, 状態変更関数 ] = React.useState( 状態の初期値を指定 );」のように書く。現在の stateの値（状態変数）と、それを更新するための関数（状態変更関数）とをペアにして返す。
  • たとえ内容が極小でも、慣習として onClick={...}内に関数は書かないほうが良いらしい。関数を外に書く場合の書き方を参考にされたし。

コード

HTML


<div id="root" class="demoRoot"></div>

jsx（関数を中に書く場合の書き方）


const CountUpBtn = () => {
  // const [状態変数, 状態変更関数] = React.useState(状態の初期値);
  const [count, setCount] = React.useState(0);
  return (
    <button onClick={() => setCount(count + 1)} className='countBtn'>
      {count}
    </button>
  )
}
//コンポーネントの描画
ReactDOM.render(
  <React.StrictMode>
    <CountUpBtn />
  </React.StrictMode>,
  document.getElementById('root')
);

jsx（関数を外に書く場合の書き方）


const CountUpBtn = () => {
  // const [状態変数, 状態変更関数] = React.useState(状態の初期値);
  const [count, setCount] = React.useState(0);
  const increment = () => setCount(count + 1);
  return (
    <button onClick={increment} className='countBtn'>
      {count}
    </button>
  )
}
//コンポーネントの描画
ReactDOM.render(
  <React.StrictMode>
    <CountUpBtn />
  </React.StrictMode>,
  document.getElementById('root')
);

編集後記

Tips

  • useState()を定義する場合は、Function Component内のトップレベルで宣言しなければならない。うっかり Function Component内に関数を書き、その中で定義しようとするとエラーが発生する。
  • useState()は、繰り返される再レンダーの間も Reactによって保持される。
  • Class Componentにおける this.setStateと似ているが、新しい stateが古いものとマージされないという違いがある。
  • 複数の state（状態）を定義したい場合、更新のタイミングに関連性がなければ、useState()は別々に宣言するほうがいい。

あとがき

  • なるほど、Function Componentで書く人が増える理由が分かる気がした。とてもシンプルに書ける。「super(props);」を書いたり、thisが指し示す対象を意識しなければいけなかったり、bindが必要だとか、Class Componentはいろいろとやることが多い。必要性がないかぎり、Class Componentで書くことはないだろう。
  • 見た印象だけでも大きく違うが、それだけではないらしい。例えば、Class Componentで書くと、ライフサイクルメソッドにしばられることで、無関係なロジックが混在して再利用しがたいというような問題が起きやすくなるそうだ。複雑なアプリならとくに可能性は増す。本来あるべき Reactの「コンポーネント」という概念に反して、適したロジックで分割しにくくなるのであれば問題だ。分割しやすい Function Componentが主流の書き方になるのは当然の結果なのかもしれない。
  • 端的に書けば、Class Componentが有する機能を、Function Componentでも書けるようにしたものが Hooksである。Hooksは、Function Component内から Reactの機能に「接続する (hook into)」ための関数。
