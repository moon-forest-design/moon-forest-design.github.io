【GatsbyJS入門】
head要素を設定する方法

概要

要点

  1. プラグイン gatsby-plugin-react-helmetと react-helmetをインストールする。
  2. gatsby-config.jsに、以下の指定を追加。
    • siteMetadata
    • gatsby-plugin-react-helmet
  3. components下に seo.jsファイルを作成。
    • react-helmet、@reach/router、prop-typesをインポートする。react-helmetは head要素を差し込む。@reach/routerは各ページのパスを取得する。prop-typesは型チェックに使用。
    • graphqlではサイト全体、もしくはトップページで使用するデータを取得する。引数からは各ページの個別のデータを取得して展開する。
    • 一度オブジェクト seoで必要な形に整形し、さらに helmet要素内で title要素や meta要素として出力（return）している。
    • defaultPropsは初期値設定に使用。
  4. 各ページコンポーネント（index.jsなど）から引数を渡して、seo.jsを呼び出す。

手順

コマンドライン


npm install gatsby-plugin-react-helmet react-helmet

gatsby-config.js


module.exports = {
  /* Your site config here */
  siteMetadata: {
    siteName: "（Webサイト名）",
    siteSubTitle: "（トップページのサブタイトル）",
    siteDescription: "（トップページの説明）",
    siteUrl: "（トップページのURL　※最後の「/」は不要）",
    siteOgpImage: "（OGP画像）",
    siteAuthor: "（著作権利者名）",
  },
  plugins: [
    "gatsby-plugin-image",
    "gatsby-plugin-sharp",
    {
      resolve: `gatsby-source-filesystem`,
      options: {
        name: `blog`,
        path: `${__dirname}/content`,
      },
    },
    {
      resolve: `gatsby-source-filesystem`,
      options: {
        name: `images`,
        path: `${__dirname}/src/img`,
      },
    },
    "gatsby-plugin-mdx",
    "gatsby-transformer-sharp",
    "gatsby-plugin-react-helmet",
  ],
}

seo.js


import * as React from 'react';
import { Helmet } from "react-helmet";
import { useStaticQuery, graphql } from "gatsby";
import { useLocation } from "@reach/router";
import PropTypes from "prop-types";

const SEO = ({ pageTitle, pageDescription, imgPath, topPage }) => {
  const { pathname } = useLocation();

  const { site } = useStaticQuery(query);
  const {
    siteName,
    siteSubTitle,
    siteDescription,
    siteUrl,
    siteOgpImage,
    siteAuthor,
  } = site.siteMetadata;

  const seo = {
    title: pageTitle ? `${pageTitle} | ${siteName}` : `${siteName} | ${siteSubTitle}`,
    description: pageDescription || siteDescription,
    image: `${siteUrl}${imgPath || siteOgpImage}`,
    url: pathname ? `${siteUrl}${pathname}` : `${siteUrl}`,
    pageType: topPage ? "website" : "article",
  };

  return (
    <Helmet>
      <html lang="ja-JP" />
      <title>{seo.title}</title>
      <meta name="description" content={seo.description} />
      <meta name="image" content={seo.image} />
      <meta name="author" content={siteAuthor} />
      <link rel="canonical" href={seo.url} />
      <meta property="og:locale" content="ja-JP" />
      {<meta property="og:site_name" content={siteName} />}
      {seo.url && <meta property="og:url" content={seo.url} />}
      {seo.pageType && <meta property="og:type" content={seo.pageType} />}
      {seo.title && <meta property="og:title" content={seo.title} />}
      {seo.description && <meta property="og:description" content={seo.description} />}
      {seo.image && <meta property="og:image" content={seo.image} />}
      <meta name="twitter:card" content="summary_large_image" />
      {seo.title && <meta name="twitter:title" content={seo.title} />}
      {seo.description && <meta name="twitter:description" content={seo.description} />}
      {seo.image && <meta name="twitter:image" content={seo.image} />}
    </Helmet>
  )
};

export default SEO;

const query = graphql`
  query {
    site {
      siteMetadata {
        siteName
        siteSubTitle
        siteDescription
        siteUrl
        siteOgpImage
        siteAuthor
      }
    }
  }
`;

SEO.defaultProps = {
  pageTitle: null,
  pageDescription: null,
  imgPath: null,
  topPage: false,
};

SEO.propTypes = {
  siteName: PropTypes.string,
  siteSubTitle: PropTypes.string,
  siteDescription: PropTypes.string,
  siteUrl: PropTypes.string,
  siteOgpImage: PropTypes.string,
  siteAuthor: PropTypes.string,
  pageTitle: PropTypes.string,
  pageDescription: PropTypes.string,
  imgPath: PropTypes.string,
  topPage: PropTypes.bool,
};

index.js


import * as React from 'react'
import Layout from '../components/layout'
import Seo from '../components/seo';

const IndexPage = () => {
  return (
    <Layout pageTitle="Home Page">

    </Layout>

export default IndexPage

編集後記

あとがき

  • seo.jsは、「SEOコンポーネントの追加」のコードを参考にして手を加えた。
    • localeを追加した。「lang="ja_JP"」で指定すると Warningエラーが出る。「lang="ja-JP"」にするほうが良いようだ。
    • siteSubTitleを追加した。title要素は、トップページでは「サイト名｜サブタイトル」となり、各ページは「ページタイトル｜サイト名」となる。
    • authorを追加した。
    • urlは、pathnameがあるか確認し、あれば付加する形にした。
    • titleTemplateは意図通りにならないので使用しないことにした。
    • link要素（rel="canonical"）を追加した。
    • meta要素（property="og:site_name"）を追加した。
    • meta要素（property="og:type"）は、トップページを「website」、各ページを「article」とした。
    • twitterUsernameは削除した。
  • React.PropTypesは、React v15.5から別パッケージになった。使いたい場合は、prop-typesライブラリをインストールしなければならない。しかし、インストールしていないが使えている。なぜでしょう。「npm list」コマンドで確認したところ、すでに入っているように見える。GatsbyJSに内包されているのかな。少し検索しただけでは情報見つからず。
  • useLocation()から代入するパス名は「pathname」にしなければいけないようだ。例えば「pathName」にすると undefinedになってしまう。
