【React入門】
buttonイベント基礎

デモ

コード概要

要点

  • ボタンを 2つ作成した。同じ仕様のボタンだが、1つ目は onClick属性内に直接処理を書き、2つ目は onClick属性内に処理を書かずに、コールバック関数として外に書いた場合のサンプルである。
  • Reactのイベント処理は、onを頭文字としたの属性で指定する。（例：onClick属性、onChange属性など）
  • 例えば、onClick={...}のように書き、中には関数を指定する。関数内には、イベントハンドラを書く。イベントハンドラとは、ユーザの操作により発生した要件に対して処理すべき命令群のこと。今回のサンプルでは、ボタンがクリックされたら「クリックされた」とコンソール表示されるようにアロー関数で書いている。

コード

HTML


<!-- onClick属性内に直接処理を書いた場合のコード -->
<div id="root" class="demoRoot"></div>

<!-- onClick属性外に処理を書いた場合のコード -->
<div id="root2" class="demoRoot"></div>

jsx


<!-- onClick属性内に直接処理を書いた場合のコード -->
const ClickBtn = () => {
  return (
    <div>
      <button onClick={() => console.log('clicked 1')}>Click me!</button>
    </div>
  )
}
ReactDOM.render(
  <ClickBtn />,
  document.getElementById('root')
);

<!-- onClick属性外に処理を書いた場合のコード -->
const ClickBtn2 = () => {
  // コールバック関数
  const handleBtn= () => {
    // イベントハンドラを書く
    console.log('clicked 2')
  }
  return (
    <div>
      <button onClick={handleBtn}>Click me!</button>
    </div>
  )
}
ReactDOM.render(
  <ClickBtn2 />,
  document.getElementById('root2')
);

編集後記

あとがき

  • Reactでは、イベントを表す propsには on[Event] という名前、イベントを処理するメソッドには handle[Event]という名前を付けるのが慣習となっている。
  • より実践的なコードは、どのような場面で使用する技術なのかイメージしやすく分かりやすいが、いかんせん内容が複雑になりがちだ。今回のサンプルのように、シンプルなコードも見て慣れておくと理解しやすくなる。
