ファイル名の表示方法

概要

要点

  1. プロジェクトディレクトリの最上位（gatsby-config.jsなどがある場所）に、blogというディレクトリを作成する。
  2. blogディレクトリ内に、拡張子.mdxのファイルを、3つ作成する。（例：post1.mdx、post2.mdx、post3.mdx）
  3. ソースプラグイン gatsby-source-filesystemがインストールされているか確認する。確認の方法は、package-lock.jsonを見ればいいのかな？　インストールしていない場合は、インストールする。おそらく必要になるであろうプラグイン gatsby-plugin-imageとあわせてインストールしてしまうといい。
  4. gatsby-config.jsファイルに、gatsby-source-filesystemを利用して blogディレクトリに関する設定を追加する。
  5. GraphiQLインターフェースで確認。エクスプローラーで allFileの nodesの nameにチェックし実行ボタンを押す。上述にて作成したmdxファイル名が表示されれば OK。
  6. 例えば、src/pages/下に blog.jsを作成し、次のようにする。
    • graphqlを importする。
    • {node.name}をファイルの数だけ繰り返し記述するために Array.mapを使用する。
    • GraphQLのデータを定数 queryへ代入する。

手順

gatsby-config.js


// gatsby-config.jsファイルに設定を追加
module.exports = {
  siteMetadata: {
    title: "サイト名",
  },
  plugins: [
    "gatsby-plugin-image",
    "gatsby-plugin-sharp",
    {
      resolve: "gatsby-source-filesystem",
      options: {
        name: `blog`,
        path: `${__dirname}/blog`,
      }
    },
  ],
};

blog.js


import * as React from 'react'
import { graphql } from 'gatsby'
import Layout from '../components/layout'

const BlogPage = () => {
  return (
    <Layout pageTitle="My Blog Posts">
      <ul>
        {
          data.allFile.nodes.map(node => (
            <li key={node.name}>
              {node.name}
            </li>
          ))
        }
      </ul>
    </Layout>
  )
}

export const query = graphql`
  query {
    allFile {
      nodes {
        name
      }
    }
  }
`

export default BlogPage
