【React Hooks】
useEffect()とは？

デモ

コード概要

要点

  • 今回は useEffect()を試す。React公式によると、useEffect()は、Class Componentにおける「ライフサイクルメソッド」を Function Componentでも使えるようにしたものなのだそうだ。useEffect()を使うと、例えば次のようなメソッドを、Function Componentでも同様の目的で使えるようになる。
    • componentDidMount
    • componentDidUpdate
    • componentWillUnmount
  • 公式「フック早わかり – React」に掲載されているコードを参考にして試してみた。useEffect()により、ページタイトルを強制的に変更する仕様である。また、ボタンをクリックすると、ページタイトル部分は動的に更新される。
  • useEffect()は次のような形で指定する。「useEffect( コールバック関数 [, 依存関係配列 ] )」
    • 第2引数を空指定「[]」すると、初回のレンダリング時のみ実行される。componentDidMountのような扱いができる。
    • 第2引数を省略すると、レンダーのたびに毎回実行される。依存関係配列を指定すれば、配列に格納された値が変更された場合のみ実行される。無駄なく処理ができる。これは componentDidUpdateのような扱いとなる。

コード

HTML


<div id="root" class="reactRoot"></div>

JavaScript（jsxファイル）


  const CountUpBtn = () => {
    // const [ 状態変数, 状態変更関数 ] = React.useState( 状態の初期値 );
    const [count, changeCount] = React.useState(0);
    // React.useEffect( コールバック関数 [, 依存関係配列 ] );
    React.useEffect(() => {
      // クリックのたびにドキュメントのタイトル部分を更新する
      document.title = `You clicked ${count} times`;
    // 依存関係配列
    }, [count]);
    return (
      <button onClick={() => changeCount(count + 1)} className='countBtn'>
        {count}
      </button>
    )
  }
  //コンポーネントの描画
  ReactDOM.render(
    <React.StrictMode>
      <CountUpBtn />
    </React.StrictMode>,
    document.getElementById('root')
  );

編集後記

補足

  • React界隈では、ライフサイクルメソッド内で行うような操作を「副作用（side-effects）」と呼ぶ。例えば、Fetchによる外部データの取得・購読、あるいは手動での DOM更新などの操作のことだ。それゆえに、副作用を操作できる useEffect()は、「副作用フック」と呼ばれる。
  • デフォルトでは、毎回のレンダー後にこの「副作用関数 useEffect()」が呼び出される。第2引数の設定や return文を追加することで実行したいタイミングを変更することができる。

あとがき

  • Class Componentの componentWillUnmount内に書いていたコードを、Function Componentで記述したい場合（クリーンアップが必要な場合）は、useEffect()内に「return () => { ... };を記述してあげれば良いようだ。それだけでうまいこと認識してくれる。詳しくは次回。
