コード概要

要点

今回は、ドロワーメニューです。ハンバーガーボタンをクリックすると、画面外からメニューがスライド表示されます。ポイントは以下ようです。

  • ハンバーガーボタン（.burgerBox）は、ヘッダーの右端に「position: fixed;」で固定表示しています。
  • ドロワーメニュー（.navList）は、画面の左側から引き出す形です。「position: absolute;」で画面外に配置していたものが、ハンバーガーボタンをクリックすることで画面内に移動します。
  • ドロワーメニュー（.navList）のリンクをクリックすれば、ページ内のリンク先へ移動します。
  • div要素（.mask）は、画面の暗転に使用しています。
  • 展開しているドロワーメニュー（.navList）が閉じるのは、ハンバーガーボタン（.burgerBox）、div要素（.mask）、もしくはドロワーメニューのリンクをクリックした場合です。

jQuery


$(function() {
  $('.burgerMenu, .mask').on('click', function() {
    $('.burgerBtn').toggleClass('open');
    $('.navList').toggleClass('open');
    $('.mask').toggleClass('open');

  });
  $('.navList__link').on('click', function() {
    $('.burgerBtn').removeClass('open');
    $('.navList').removeClass('open');
    $('.mask').removeClass('open');
  });
});

編集後記

あとがき

  • ドロワーメニューのデメリットとしては、メニュー項目を選択するまでのクリック回数が増えてしまうこと。それから、普段見えていないものは、記憶してもらいにくいということなどが挙げられる。定番化しているからといって安易に導入しないよう気をつける必要がある。
