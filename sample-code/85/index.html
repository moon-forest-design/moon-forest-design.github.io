React Hooks入門「useEffect」

コード概要

要点

  • 今回は、以前 Class Componentで作成したカウントアップボタンを、Function Componentで書き直してみた。Class Componentではいわゆる「State」を使用。Function Componentでは代わりに、React Hooksの useStateを使う。
  • Reactのコードを外部ファイルに書き出して、React Hooksを使用する場合は、importやexportで指定する必要があるはずなのだが、CDNで試した分ではとくに何もする必要がなかった。バージョンによる違いだろうか。不明。
  • useStateの定義は次のように書く。「const [状態変数, 状態変更関数] = React.useState(状態の初期値);」
  • return()内の<>と</>は、これ何だろうね。これまでは div要素を書くことが多かったようだが、こういう書き方もあるのか。どうやって調べたら（検索したら）いいんだ。
  • 。

コード

HTML


<div id="root" class="reactRoot"></div>
<script type="text/babel">
  const CountUpBtn = () => {
    // const [状態変数, 状態変更関数] = React.useState(状態の初期値);
    const [count, changeCount] = React.useState(0);
    return (
      // 何だこれ？？？
      <>
        <button onClick={() => changeCount(count + 1)} className='countBtn'>
          {count}
        </button>
      </>
    )
  }
  //コンポーネントの描画
  ReactDOM.render(
    <React.StrictMode>
      <CountUpBtn />
    </React.StrictMode>,
    document.getElementById('root')
  );
</script>

編集後記

あとがき

  • なるほど、Function Componentで書く人が増えた理由が分かる気がした。とてもシンプルに書ける。「super(props);」を書いたり、thisが指し示す対象を意識しなければいけなかったり、bindが必要だとか、Class Componentはいろいろとやることが多い。必要がないかぎり、Class Componentで書くことはないだろう。
  • React Hooksを定義する場合は、Function Component内のトップレベルで宣言しなければならない。うっかり Function Component内に関数を書き、その中で定義しようとすることもあるかもしれない。気をつけよう。
