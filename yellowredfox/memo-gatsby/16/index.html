tagsページを設置する方法

概要

要点

公式の「Creating Tags Pages for Blog Posts | Gatsby」を参考にして設置した。公式は MD版なので、MDX用に書き直した。

  1. Markdownファイルにタグを追加する。例：tags: ["animals", "Chicago", "zoos"]
  2. GraphiQLでクエリを試す。（http://localhost:8000/___graphql）
  3. タグページテンプレートを作成する。src下にtemplatesというディレクトリを作成し、そこに tags.jsファイルを作成する。公式で紹介されているコードを丸っと参考にする。ただし、MDXファイルを使用するのであれば、allMarkdownRemarkを allMdxと書き換える必要がある。
  4. プロジェクトディレクトリ直下に、gatsby-node.jsファイルを作成する。gatsby-node.jsの内容も、上述同様に修正する。また公式のチュートリアル通りに作っているのなら、さらに「path.resolve("src/templates/blog.js")」を「path.resolve("src/pages/blog/{mdx.slug}.js")」に変更する必要があるかもしれない。つまり、{mdx.slug}.jsの場所を指定する必要がある。
  5. タグインデックスページを作成する。src/pages/下に tags.jsファイルを作成する。これは、allMarkdownRemarkを、allMdxと書き換えるだけで良さそうだ。
  6. 「http://localhost:8000/tags/」へアクセスして、表示されればとりあえずは OK。あとは、layout.jsや、CSSを設定するだけ。

手順

MDXファイル


---
title: "A Trip To the Zoo"
tags: ["animals", "Chicago", "zoos"]
---
I went to the zoo today. It was terrible.

GraphiQLクエリ


query MyQuery {
  allMdx {
    totalCount
    edges {
      node {
        slug
        frontmatter {
          title
          tags
        }
      }
    }
  }
}

src/templates/tags.js


import React from "react"
import PropTypes from "prop-types"
import { Link, graphql } from "gatsby"
const Tags = ({ pageContext, data }) => {
  const { tag } = pageContext
  const { edges, totalCount } = data.allMdx
  const tagHeader = `${totalCount} post${
    totalCount === 1 ? "" : "s"
  } tagged with "${tag}"`
  return (
    <div>
      <h1>{tagHeader}</h1>
      <ul>
        {edges.map(({ node }) => {
          const slug = node.slug
          const { title } = node.frontmatter
          return (
            <li key={slug}>
              <Link to={`${slug}`}>{title}</Link>
            </li>
          )
        })}
      </ul>
      <Link to="/tags">All tags</Link>
    </div>
  )
}
Tags.propTypes = {
  pageContext: PropTypes.shape({
    tag: PropTypes.string.isRequired,
  }),
  data: PropTypes.shape({
    allMdx: PropTypes.shape({
      totalCount: PropTypes.number.isRequired,
      edges: PropTypes.arrayOf(
        PropTypes.shape({
          node: PropTypes.shape({
            frontmatter: PropTypes.shape({
              title: PropTypes.string.isRequired,
            }),
            slug: PropTypes.string.isRequired,
          }),
        }).isRequired
      ),
    }),
  }),
}
export default Tags
export const pageQuery = graphql`
  query($tag: String) {
    allMdx(
      limit: 2000
      sort: { fields: [frontmatter___date], order: DESC }
      filter: { frontmatter: { tags: { in: [$tag] } } }
    ) {
      totalCount
      edges {
        node {
          slug
          frontmatter {
            title
          }
        }
      }
    }
  }
`

gatsby-node.js


const path = require("path")
const _ = require("lodash")
exports.createPages = async ({ actions, graphql, reporter }) => {
  const { createPage } = actions
  const blogPostTemplate = path.resolve("src/pages/blog/{mdx.slug}.js")
  const tagTemplate = path.resolve("src/templates/tags.js")
  const result = await graphql(`
    {
      postsRemark: allMdx(
        sort: { order: DESC, fields: [frontmatter___date] }
        limit: 2000
      ) {
        edges {
          node {
            slug
            frontmatter {
              tags
            }
          }
        }
      }
      tagsGroup: allMdx(limit: 2000) {
        group(field: frontmatter___tags) {
          fieldValue
        }
      }
    }
  `)
  if (result.errors) {
    reporter.panicOnBuild(`Error while running GraphQL query.`)
    return
  }
  const posts = result.data.postsRemark.edges
  posts.forEach(({ node }) => {
    createPage({
      path: node.slug,
      component: blogPostTemplate,
    })
  })
  const tags = result.data.tagsGroup.group
    tags.forEach(tag => {
    createPage({
      path: `/tags/${_.kebabCase(tag.fieldValue)}/`,
      component: tagTemplate,
      context: {
        tag: tag.fieldValue,
      },
    })
  })
}

src/pages/tags.js


import React from "react"
import PropTypes from "prop-types"
import kebabCase from "lodash/kebabCase"
import { Helmet } from "react-helmet"
import { Link, graphql } from "gatsby"
const TagsPage = ({
  data: {
    allMdx: { group },
    site: {
      siteMetadata: { title },
    },
  },
}) => (
  <div>
    <Helmet title={title} />
    <div>
      <h1>Tags</h1>
      <ul>
        {group.map(tag => (
          <li key={tag.fieldValue}>
            <Link to={`/tags/${kebabCase(tag.fieldValue)}/`}>
              {tag.fieldValue} ({tag.totalCount})
            </Link>
          </li>
        ))}
      </ul>
    </div>
  </div>
)
TagsPage.propTypes = {
  data: PropTypes.shape({
    allMdx: PropTypes.shape({
      group: PropTypes.arrayOf(
        PropTypes.shape({
          fieldValue: PropTypes.string.isRequired,
          totalCount: PropTypes.number.isRequired,
        }).isRequired
      ),
    }),
    site: PropTypes.shape({
      siteMetadata: PropTypes.shape({
        title: PropTypes.string.isRequired,
      }),
    }),
  }),
}
export default TagsPage
export const pageQuery = graphql`
  query {
    site {
      siteMetadata {
        title
      }
    }
    allMdx(limit: 2000) {
      group(field: frontmatter___tags) {
        fieldValue
        totalCount
      }
    }
  }
`

編集後記

あとがき

  • layout.js、もしくは header.jsなどで helmetを設定している場合は、「import { Helmet } from "react-helmet"」などはいらなくなる。
  • クエリの slugは、いろいろな箇所にあるようだ。現状利用する上での細かな違いは分からないけれど、とりあえず、前に「/（スラッシュ）」が付くもの、付かないものがあるので注意が必要。例えば、ドメイン等と結合して使う場合に、コーディングを誤ると「//（スラスラ）」になる可能性がある。
    • edges - node - slug
    • edges - node - fields - slug
    • nodes - slug
    • nodes - fields - slug
  • まれに、fieldsが表示される場合とされない場合がある。なぜなのか分からないでいたが、gatsby-node.js内の createPage()で指定しているのだと分かった。上述のいろいろな箇所に slugが発生するのは、blogや tag、ページネーションなどのページを動的に作成するために、誤って不要な createPage()を複数行っていたようだ。
  • どちらも tags.jsというファイル名なので混乱するかもしれない。src/templates/tagTemplate.js、src/pages/tag-list.jsとしたほうが分かり良いかもしれない。
