【React入門】
props.children

デモ

コード概要

要点

  • React公式の「コンポジション vs 継承」で紹介されているコード。Reactでコンポーネントを再利用するときは、「コンポジション」と呼ばれるやり方がおすすめらしい。
  • 今回の事例では、例えばブログの sidebarなどのように事前に何が入るか分からないときに使える「props.children」を紹介している。コンポーネントをどのように分割し、組み合わせて構成（コンポジション）するのか参考になるだろう。以下、要点を記述。
    • 関数 WelcomeDialog()にて、関数 FancyBorder()の呼び出しを行っている。おもしろいのは、<FancyBorder />ではないところだ。中に h3要素や p要素が含まれている。これらの<FancyBorder>内に記述されている子要素も引数として渡しているのだそうだ。そして、関数 FancyBorder()で propsとして受け取ることができる。{props.children}と記述することで受け取った要素を構成し直して返す形だ。
    • 関数 FancyBorder()で返す div要素に対しては、ある程度決まった CSSがつけられている。さらに borderの色を指定することができる。つまり、明確に決まっている部分は指定しておき、「あとはご自由に」という形で柔軟な対応もできる汎用的な構成。

コード

HTML


<div id="root" class="demoRoot"></div>

jsx


  // fancy：事前に明確な決まりがない
  function FancyBorder(props) {
    return (
      <div className={'FancyBorder FancyBorder-' + props.color}>
        {props.children}
      </div>
    );
  }
  // <FancyBorder>～</FancyBorder>内の要素を引数として渡している
  function WelcomeDialog() {
    return (
      <FancyBorder color="blue">
        <h3 className="Dialog-title">
          Welcome
        </h3>
        <p className="Dialog-message">
          Thank you for visiting our spacecraft!
        </p>
      </FancyBorder>
    );
  }
  ReactDOM.render(
    <React.StrictMode>
      <WelcomeDialog />
    </React.StrictMode>,
    document.getElementById('root')
  );

編集後記

あとがき

  • 継承については、とくに書かれていない。オブジェクト指向の inheritanceみたいな話だろうか。継承による階層構造が推奨されるケースはあまりないようなので、とりあえず今のところは気にしなくて良いのかもしれない。
