【React入門】
コンポーネントのいろいろな書き方

デモ

コード概要

要点

  • Reactのコンポーネントと引数を使う場合には、いろいろな書き方がある。コンポーネントには、Class Componentと Function Componentがある。引数には、propsで受け取る方法もあるし、propsと記述せずに分割代入で書く方法もある。
  • おおよそ関数（Function Component）で書く方法が良さそう。とくに 4、もしくは 5のように仮引数を分割代入で受け取る書き方なら、シンプルでより良い印象。

コード

HTML

jsx


  // 1　関数 functionを使い、仮引数を propsで受け取る方法
  function Component1( props ) {
    return (
      <div>コンポーネント1： {props.way}</div>
    );
  }

  // 2　アロー関数を使い、仮引数を propsで受け取る方法
  const Component2 = ( props ) => (
    <div>コンポーネント2： {props.way}</div>
  );

  // 3　アロー関数を使い、仮引数を propsで受け取る方法（returnあり）
  const Component3 = ( props ) => {
    // ※：ここに何かしらの処理が入る場合にreturnが必要になる
    return (
      <div>コンポーネント3： {props.way}</div>
    );
  };

  // 4　関数 functionを使い、仮引数を分割代入で受け取る方法
  function Component4({ way }) {
    return (
      <div>コンポーネント4： {way}</div>
    );
  }

  // 5　アロー関数を使い、仮引数を分割代入で受け取る方法
  const Component5 = ({ way }) => (
    <div>コンポーネント5： {way}</div>
  );

  // 6　class構文を使い、引数は this.props.***で受け取る方法
  class Component6 extends React.Component {
    render() {
      return (
        <div>コンポーネント6： {this.props.way}</div>
      );
    }
  }

  // 7　React.createClass()を使う方法
  // ※：2017年4月リリースのReact 15.5で、使用警告メッセージ
  // ※：2017年9月リリースのReact 16で、正式に廃止になった
  // var Component7 = React.createClass({
  //   render: function() {
  //     return (
  //       <div>コンポーネント7： {this.props.way}</div>
  //     );
  //   }
  // });

  const render = () => ReactDOM.render(
    <React.StrictMode>
      <Component1 way="関数 functionを使い、仮引数を propsで受け取る方法" />
      <Component2 way="アロー関数を使い、仮引数を propsで受け取る方法" />
      <Component3 way="アロー関数を使い、仮引数を propsで受け取る方法（returnあり）" />
      <Component4 way="関数 functionを使い、仮引数を分割代入で受け取る方法" />
      <Component5 way="アロー関数を使い、仮引数を分割代入で受け取る方法" />
      <Component6 way="class構文を使い、引数は this.props.***で受け取る方法" />
      { /* <Component7 way="React.createClass()を使う方法" /> */ }
    </React.StrictMode>,
    document.getElementById('root')
  );
  render();

編集後記

あとがき

  • 現状、コンポーネントは関数で書くのが一般的には主流とのこと。
  • これまではライフサイクルメソッドが必要なときには、class構文（Class Component）で書く必要があった。しかし、React Hooksという APIを使えばその必要はない。例えば、Stateを使いたければ、React Hooksの useStateがある。React 16.8以降であれば、関数（Function Component）＋ React Hooksで良いみたい。
  • 古いコードを見たときに、class構文（Class Component）にお目にかかることはあるかもしれない。現状においては残念ながら、知らずとも良いとはならないだろう。しかし、class構文で書くときに extendsする React.Componentが将来的に廃止になる予定らしいので、目にする機会は少なくなっていくのかもしれない。
