React概要

React基礎

React
JavaScriptライブラリ。UIとデータを分離しやすくする利点がある。関心の分離（separation of concerns）が捗る。
JSX
JavaScriptの拡張構文。JSXを使うと、JavaScriptのコードの中に HTMLのようなタグを直接記述することができる。Reactでコードを書く場合は JSXを使うのが一般的。ただし、Babelなどでコンパイルする必要がある。
フラグメント
Reactで複数の要素を returnしたいときは、div要素などで囲ってあげる必要があるのだが、div要素を使うのが不適切なときは、フラグメント（<React.Fragment>）で囲んであげると良い。短縮記法（<>）もある。
Reactコンポーネント
再利用しやすくするための部品化。UIが同じで、内容（データ）は違う場合に対応しやすくなる。書き方には Function Component、及び Class Componentがある。React Hooksの登場により、Function Componentが主流となった。
Props
Propsオブジェクト。コンポーネント間のデータの受け渡しに使用する引数。propertyの略称。
State
レンダー後に変更される「状態（state）」を示すデータのこと。例えば、時刻を表示するとした場合の数値は、Stateとして設定する。コードを Function Componentで書くのであれば、React Hooksの useStateを使用する。
ライフサイクル
Reactの処理体系を Mounting、Updating、Unmountingの 3段階に分けて考えるときの概念。Class Componentで書く場合においては、componentDidMount()、shouldComponentUpdate()、componentDidUpdate()、componentWillUnmount()などのライフサイクルメソッドを使って書く。

React Hooks

useState
関数コンポーネント内で「状態（state）」を管理する。
useEffect / useLayoutEffect
useEffectは、「副作用」として実行したい処理を書く。useLayoutEffectは、レンダーが画面に反映される手前で実行したい処理を書く。
useRef
DOMに対する参照を保持する。
useContext
コンテキストからデータを取り出す。
useReducer
現在の State値を受け取って新しい State値を返す。
useCallback
メモ化された関数を返す。
useMemo
メモ化された値を返す。
カスタムフック
React hooksの処理をコンポーネントに直接書かず、別ファイルに切り出して新たな hooksとして定義した関数。

Reactその他

React Router
ルーティングできるようにする機能。SPAにおいて、UIと URLを対応づけることで、ブラウザの履歴やブックマーク、戻る・進むボタンなどの問題を解決する。
Suspense
コンポーネントが何かしらを「待機」できるようにする機能。（※：実験段階）

React開発環境

Reactお試し環境

まだ入門レベルの段階にあり、Reactを試してみたいだけであれば、次の方法が有用。

CDNの利用
例えば unpkgなどを使えば、いつものローカル環境で試すことができるので気楽にはじめられる。サーバ環境が必要であれば、Live Serverなどを入れるといい。
Code Playgroundの利用
Web上で気軽にコードを試すことができるので便利。しかし、慣れたテキストエディタのように打ち込むことができないので小さなストレスがたまり続ける。また、アカウント登録が必要なサイトもあるので、面倒くさいと感じるかもしれない。JSFiddle、CodeSandbox、codePenなど。

React開発環境（簡便な方法）

簡便に Reactの開発がはじめられるツール。webpackなどの設定の手間を省くことができるため、速やかに開発環境が構築できる。中身は 設定済みの webpack、Babel、ESLintなどが含まれている。

Create React App
新しい SPAを作りたいときに役立つツール。Node.jsを導入し、インストールするだけで OK。
Next.js
サーバサイドで開発したいときに役立つツール。Zeit社が開発した。（※：後にVercel社に改名）
Gatsby.js
コンテンツ中心のブログなどを作りたいときに役立つツール。静的サイトジェネレータ。

React開発環境（ゼロから組み合わせる方法）

次のようなものの中から必要なツールを選び、組み合わせてツールチェインとして利用する方法。

npm
定番のパッケージマネージャ。
yarn
人気があるパッケージマネージャ。
webpack
定番のモジュールバンドラ。
Parcel
モジュールバンドラ。
Babel
定番のコンパイラ。

型チェック

TypeScript
Microsoft社が開発した JavaScriptに型注釈等の機能を追加したスーパーセット。
Flow
Facebook社が開発した JavaScript静的型チェッカー。
PropTypes
React標準の型チェックの方法として使われてきた。TypeScriptや Flowの登場により、影が薄くなった。

その他のツール

その他、状況に応じて利用すると捗るツール。

React Developer Tools
ブラウザの拡張アドオン。Reactによる開発時のデバッグに利用される。
Browserify
JavaScriptにはないモジュール機能が使いたいときに利用。
Prepack
JavaScriptを高速化したい場合。

React関連用語

SPA
シングルページアプリケーション（single page application）の略称。画面遷移を必要とする状況において、ページ全体をロードせず、一部のみロードする仕組みで作られる。
レンダー
レンダー（render）は、描画すること。
ツールチェイン
案件に必要となるツールを組み合わせて作る開発環境のこと。パッケージマネージャやモジュールバンドラ、コンパイラなどを組み合わせて構築する。また、すでに組み合わせて公開されているものもある。（例：Create React App、Next.js、Gatsby.jsなど）
パッケージマネージャ
ソフトウェアのインストールやアンインストールを管理するツール。（例：npm、yarn）
モジュールバンドラ
複数のファイルを 1つにまとめて出力してくれるツール。例えば、JavaScriptやCSSなど、をひとまとめにし、ミニファイ（余計なコードを取り除くこと）もしてくれるので、表示速度を向上させる利点がある。（例：webpack）
コンパイラ
コンピュータなどが理解できる形に翻訳するツール。多くの場合、人間が理解しやすいプログラミング言語から、コンピュータが理解しやすい機械語へ変換するツールを指す。また、バージョンのダウングレードなどの変換を指すこともある。（例：Babel）
コンテキスト
ルートコンポートでのみ Stateを保持するやり方では破綻する規模のアプリケーションを作成する場合は、コンテキストを利用する。中継する必要がなくなる。
GraphQL
Web APIを宣言的に記述するための手法。
テスト駆動開発
実装よりもテストを先行させる開発手法。Jestなどを利用。
HOC
higher-order componentの略称。高階コンポーネントのこと。
