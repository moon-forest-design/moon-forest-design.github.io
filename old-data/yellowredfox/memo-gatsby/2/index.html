GatsbyJSのインストール

概要

要点

  1. 事前に、コマンドラインツールとNode.jsをインストールしておく。
  2. 右クリックをして管理者権限で Git Bashを起動する。sudoなどが使える場合は、標準起動でも問題ない。Windowsであればおそらく sudoは使えないので、状況によって臨機応変に対応されたし。
  3. GatsbyJSの CLIツールをインストールする。CLI（コマンドラインインタフェース）をインストールすると、GatsbyJS独自のコマンドを使うことができるようになる。
  4. CLIツールのインストールができたようであれば、バージョンを確認する。Ver3以降であれば OK。（2021年11月においては CLIが Ver3.14.1で、GatsbyJS本体は Ver4.1.3と出た。）
  5. Starterを利用してプロジェクトを作成する。GatsbyJSの作業場所（ディレクトリ名）を指定し、Starterは、Gatsby Starter Libraryで案件にあったものを探すといい。
  6. 利用する Starterの内容によっては、ダウンロードに時間がかかるかもしれない。
  7. ダウンロードが完了したら作成したディレクトリへ移動し、GatsbyJS開発環境を起ち上げる。
  8. ブラウザに、http://localhost:8000/ を入力して表示する。今回の場合は、「Hello World!」と表示されれば成功。
  9. Gitの対象から除外したい場合は、.gitignoreを編集。あるいは、VSCodeのソース管理にて右クリックして、「.gitignoreに追加」でも良いかもしれない。

GatsbyJSのインストール手順（Windowsの場合）

コマンドライン


// GatsbyJSの CLIツールをインストール。
$ npm install -g gatsby-cli

// CLIツールのバージョンを確認。
$ gatsby --version

// Starterを使用したプロジェクトの作成。
// gatsby new [ディレクトリ名] [StarterのURL]
$ gatsby new gatsby-test https://github.com/gatsbyjs/gatsby-starter-hello-world

// GatsbyJS開発環境の起ち上げ。
$ cd gatsby-test
$ gatsby develop

.gitignoreの編集例


#追加
.prettier*
gatsby-*.js
*.json
*.md
src/
static/

.gitコマンド例


# ファイルのキャッシュを削除する場合
git rm --cached index.jp

# ディレクトリのキャッシュを削除する場合
git rm --cached -r src/

# .gitignoreを反映させるため addして、コミットする
git add .gitignore
git commit -m ".gitignore（除外指定）を追加"

編集後記

あとがき

  • Starterを使わない方法もあるが、コマンドラインの表示がおかしくなった。Windowsと Node.jsはあまり相性が良くないらしい。残念ながら、ゼロから作りたい場合でも、Hello Worldなどの Starterを利用するしかない。
