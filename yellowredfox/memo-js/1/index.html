JavaScriptの歴史

JavaScriptに関連する主な出来事

1995年
Netscape Communications社の Brendan Eich氏が、JavaScriptを開発。名称の変遷 Mocha → LiveScript → JavaScripとなった。
1996年 11月
Netscape Communications社は、JavaScriptの標準化案を ECMAへ提出。
1997年 6月
ECMAScript 1を策定。JavaScriptのコアとなる仕様が ECMAScriptとして国際的に標準化された。
1998年 6月
ECMAScript 2を策定。
1999年 12月
ECMAScript 3を策定。正規表現、例外処理、文字列操作などの機能を追加。
2005年 2月
Google社が、Googleマップをリリース。Ajaxのお手本のようなサービスがうまれ、JavaScriptが脚光をあびる。
2009年
ライアン・ダール氏が、Node.jsを作成。JavaScriptは、サーバサイドでも開発できる言語となった。
2009年 12月
ECMAScript 5を策定。配列、オブジェクトのメソッド、JSONサポートなどの機能を追加。
2011年 6月
ECMAScript 5.1を策定。
2015年 6月
ECMAScript 6を策定。クラス、モジュール、アロー関数、Promise、分割代入、const、let（及び静的スコープ）などの機能を追加。2015年に策定されたこともあり、ES2015と呼ばれることもある。
2016年 6月
ECMAScript 7を策定。
2017年 6月
ECMAScript 8を策定。async / awaitなどの機能を追加。
2018年 6月
ECMAScript 9を策定。オブジェクトに対するスプレッド構文などの機能を追加。
2019年 6月
ECMAScript 10を策定。
2020年 6月
ECMAScript 11を策定。

JavaScriptに関する歴史の補足

  • ECMAは、European Computer Manufacturers Association（欧州電子計算機工業会）の略称。情報通信システムなどの分野における国際的な標準化団体。1994年に同団体の名称は、Ecma Internationalへ変更された。
  • JavaScriptの言語使用の変更は、Ecma Internationalの TC-39委員会で策定される。
  • ECMAScript 4は、いろいろと混迷し策定されなかった。
