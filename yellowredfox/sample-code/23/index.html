【React入門】
Array.map()とkey

デモ

コード概要

要点

  • React公式の「リストと key」で紹介されているコード。以前書いた Reactで Array.map()を使うときのやり方は、key無し。今回は key有り。
  • Keyは、どの要素が変更・追加・削除されたのかを Reactが識別するために利用するそうだ。コードとして出力はされず、あくまでも React内での管理に利用される。
  • Key値の設定について
    • データ内に id値があればそれを流用するといい。
    • データ内に id値がなければ、indexを使う方法もあるが非推奨。
    • UNIX時間を利用する例もある。連番であることに意味はなく、一意であることが大事らしい。

コード

HTML


<div id="root" class="demoRoot"></div>

JavaScript（jsxファイル）


  const ListItem = ({work, name}) => {
    // 正しい書き方：関数で書く場合は、この li要素に key属性をつけないらしい
    return <li>{work} : {name}</li>;
  };

  const MemberList = (props) => {
    const members = props.members;
    const listItems = members.map((member) =>
      // 正しい書き方：関数を呼び出すときに key属性をつけるだけでいいらしい
      <ListItem key={member.id.toString()}
                work={member.work}
                name={member.name}
      />
    );
    return (
      <ul>
        {listItems}
      </ul>
    );
  };

  const members = [
    { id: 1, name: 'Rabbit', work: 'Web Director' },
    { id: 2, name: 'Cat', work: 'Web Designer' },
    { id: 3, name: 'Fox', work: 'Web Programmer' },
    { id: 4, name: 'Raccoon dog', work: 'Markup Engineer' },
  ];
  ReactDOM.render(
    <React.StrictMode>
      <MemberList members={members} />
    </React.StrictMode>,
    document.getElementById('root')
  );

編集後記

あとがき

状況によっては ul要素の中に直接 Array.map()のコードを記述してもかまわないが、map()内がネストしすぎるようであれば、できるだけ外に抽出して書くほうが良いそうです。
