【React入門】
class構文による時刻表示

デモ

コード概要

要点

  • 今回は、関数ではなく、クラス構文（class）で定義するやり方でコンポーネントを作成する。一般的に「Class Component」と呼ばれる。
  • Class Componentの主な特徴として、State（状態）がある。Stateは Propsと違い、値を変更できる。
  • React公式の「state とライフサイクル」で紹介されているコードを試してみよう。
    • コンポーネントを呼び出すときに、「<Clock date={new Date()} />」のようにして日付を取得して引数として渡すのではなく、classで定義したコンポーネント側で処理してもらう形にする。
    • super()は、親のコンストラクタ（React.Component）を呼び出している。コンストラクタで this（this.stateなど）を使いたい場合は、「super(props);」と書く必要がある。
    • コンストラクタで作成した this.state（状態）を、コンストラクタ以外で更新したい場合は、setState()を使う。直接代入できるのは、コンストラクタ内のみ。
    • Classが DOMとして描画されるときのことを Reactでは「マウント (mounting)」と呼ぶ。Classが生成した DOMが削除されるときのことを React では「アンマウント (unmounting)」と呼ぶ。
    • ライフサイクルメソッド componentDidMount()には、DOMに描画された直後に実行したい処理を書く。
    • ライフサイクルメソッド componentWillUnmount()には、componentDidMount()メソッドの解除処理を書く。

コード

HTML


<div id="root" class="demoRoot"></div>

jsx


  // クラス構文でコンポーネントを作成
  class Clock extends React.Component {
    constructor(props) {
      // super(props)は、this.stateを使うために必要
      super(props);
      this.state = {date: new Date()};
    }

    // componentDidMount()は、レンダー直後に実行したい処理を書く
    componentDidMount() {
      this.timerID = setInterval(
        () => this.tick(),
        1000
      );
    }

    // componentWillUnmount()は、componentDidMount()の解除処理を書く
    componentWillUnmount() {
      clearInterval(this.timerID);
    }
    
    tick() {
      this.setState({
        date: new Date()
      });
    }

    render() {
      return <p>時刻　{this.state.date.toLocaleTimeString()}</p>;
    }

  }

  ReactDOM.render(
    <React.StrictMode>
      <Clock />
    </React.StrictMode>,
    document.getElementById('root')
  );

編集後記

あとがき

  • 似たような印象があるため、Propsと Stateを比べることもあるが、とくに対比的なものではなく、「どちらが優れているか？　どちらを使うか？」という話でもない。例えば、Propsの内容を Stateの初期値として利用しても良い。
