React入門
「class構文によるコンポーネント化」

コード概要

要点

  • React入門2でも書いた「コンポーネント」と呼ばれる書き方。今回は classで定義する「クラス構文（class）」と呼ばれる方法でコンポーネントを作成する。関数の propsと似たような役割として state（状態）がある。
  • 公式 Docsの MAIN CONCEPTSの「state とライフサイクル」に、クラス構文にしたほうが良い場合の事例が掲載されている。引数を受け取る書き方ではないほうが都合が良いときもあるらしい。クラス構文で書くことによってのみ発生するメリット、あるいはクラス構文でしか成しえない処理があるということだろうか。状況に応じて使い分けることになるようだ。
  • コンポーネントを呼び出すときに、「<Clock date={new Date()} />」のようにして日付を取得して引数として渡すのではなく、classで定義したコンポーネント側で処理してもらう形にする。
  • super()は、親のコンストラクタ（React.Component）を呼び出している。コンストラクタで this（this.stateなど）を使いたい場合は、「super(props);」と書く必要がある。
  • Classが DOMとして描画されるときのことを Reactでは「マウント (mounting)」と呼ぶ。Classが生成した DOMが削除されるときのことを React では「アンマウント (unmounting)」と呼ぶ。
  • componentDidMount()メソッドには、DOMに描画された直後に実行したい処理を書く。
  • componentWillUnmount()メソッドには、componentDidMount()メソッドの解除処理を書く。

コード

HTML


<div id="root" class="reactRoot"></div>
<script type="text/babel">
  // クラス構文でコンポーネントを作成
  class Clock extends React.Component {
    constructor(props) {
      // super(props)は、this.stateを使うために必要
      super(props);
      this.state = {date: new Date()};
    }

    // componentDidMount()は、連打―直後に実行したい処理を書く
    componentDidMount() {
      this.timerID = setInterval(
        () => this.tick(),
        1000
      );
    }

    // componentWillUnmount()は、componentDidMount()の解除処理を書く
    componentWillUnmount() {
      clearInterval(this.timerID);
    }
    
    tick() {
      this.setState({
        date: new Date()
      });
    }

    render() {
      return <p>Reactテスト：コンポーネントを使う（classで定義） 時刻　{this.state.date.toLocaleTimeString()}</p>;
    }

  }

  ReactDOM.render(
    <React.StrictMode>
      <Clock />
    </React.StrictMode>,
    document.getElementById('root')
  );

</script>
