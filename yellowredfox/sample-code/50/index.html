カリー化とは？

デモ

（※：デモはコンソールに表示）

コード概要

要点

  • 関数の合成とは？　のコードを見ていて、「カリー化」について理解不足だと感じたのでおさらい。カリー化については、本「Reactハンズオンラーニング 第2版」の P54にて説明されている。カリー化とは、「その場で処理を実行せずに、関数として返却することで、処理の実行のタイミングを呼び出し元にゆだねることができる」とある。
  • しかしこの本に掲載されているコードは、カリー化の利用目的も含めたより実践に近い内容になっているためか、イメージはつかみやすいのだが、コード自体には頭がついていかない印象。そこで、意図するところを言葉で補い、簡略化してみた。おそらくこんな感じの話だ。
  • 処理の流れ。
    • STEP1：num1の値が決まった時点で関数 addに引数として渡して実行し、関数を返してもらっていったん pendingに入れておく。
    • （～この間に APIによるデータ取得待ちなどが発生する～）
    • STEP2：num2の値が決まり実行できる状態になったら、関数 pendingにて引数を num2へ渡して実行する。
  • カリー化がこういうものであるとするなら、自分の言葉で言い換えて書くと、「関数を戻り値にすることによって、できるところまで段階的に処理を進めておくことができるテクニック」と言えそうだ。
  • アロー関数で書くと簡潔に書けて良いのだが、知らない人には意図が見えにくくなるデメリットは感じる。functionで書くほうが「ああ、高階関数が関数を返しているわけね」と分かり良い。使い慣れるまでは、functionで書くほうが良いかもしれない。

コード

JavaScript（アロー関数で書いた場合）


// 高階関数が戻り値として関数を返す「カリー化」と呼ばれるテクニック

// 高階関数add
const add = num1 => num2 =>
  console.log(num1 + num2);

// STEP1：num1の値が決まった時点で関数addに引数として渡して実行し、いったんpendingに入れておく
const pending = add(1);

// ～この間にデータ取得待ちなどが発生し、実行できる常態になったらSTEP2を行う～

// STEP2：関数pendingにnum2を入れて実行する
pending(2);

JavaScript（functionで書いた場合）


function add(num1) {
  return function(num2){
    console.log(num1 + num2);
  }
}
const pending = add(1);
pending(2);

編集後記

あとがき

  • カリー化の「カリー」は、キャリーバックなどの carryの方かと思ったら、curryの方らしい。curryと言えばカレーライスなどのカレー（カリー）だ。何か別の意味があるのか。あるいは何かしらの根源的な意味がある言葉なのか。
