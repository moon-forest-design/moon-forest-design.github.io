ターミナルの
コマンドまとめ

概要

ターミナルとは？

  • コマンドと呼ばれる文字を入力することで、コンピュータに命令を行う方法。もしくはそのソフトウェア群。簡素な真っ黒い画面に入力を行う形になるため、コマンドを知らないと何もできない。同じような意味あいで、コマンドラインと呼ばれることもある。
  • ターミナル型のソフトウェア形式は CUI（Character User Interface）と呼ばれる。対して、コマンドを知らなくてもマウスなどを操作することで利用できるソフトウェア形式は GUI（Graphical User Interface）と呼ばれる。

主なコマンド

UNIX系コマンド

cd
ディレクトリ間の移動。「cd ディレクトリ名」で、移動したいディレクトリへカレントが移る。「cd」だけ打つと、ホームディレクトリへ移動。
ls
ディレクトリ内にあるファイルを確認。「ls」はファイル名だけ表示。「ls -l」はファイル名と詳細を表示。「ls -al」は先頭が .（ドット）ではじまるファイルも含めて表示。
mkdir
ディレクトリを作成。「mkdir ディレクトリ名」として作成する。
rm
ファイル、あるいはディレクトリの削除。「rm -rf ディレクトリ名」とすれば、削除確認がないので簡便。
touch
空ファイルを作成。「touch ファイル名」とすれば、新規に空のファイルを作成する。
find
ファイル、あるいはディレクトリを探す。
chmod
ファイル、あるいはディレクトリのパーミッション変更。

npmコマンド

npm -v
npmのバージョンを確認。
npm init
npmで管理するプロジェクトディレクトリを作成する。自動的に package.jsonファイルが生成される。質問にすべて Enterで応えるのが手間なら「npm init -y」とすれば簡便。
npm install
パッケージをインストールする。カレントディレクトリへ移動し、「npm install」とだけ打ち込むと package.jsonに記載されたパッケージが一括インストールされる。「npm install パッケージ名」とすれば指定したパッケージのみインストールされる。オプション「-g」をつければ、グローバルインストールになる。
npm uninstall
パッケージをアンインストールする。カレントディレクトリへ移動し、「npm uninstall パッケージ名」とすればアンインストールされる。
npm list
インストール済みのパッケージ一覧を表示する。個別に確認する場合は「npm list （パッケージ名）」とする。
npm outdated
インストール済みのパッケージより新しいバージョンがあるか確認する。
npm update
パッケージをアップデートする。「npm update パッケージ名」とすればアップデートされる。
npm cache
キャッシュに関するコマンド。パッケージ関係で原因不明のエラーが出た場合は、「npm cache clean」としてキャッシュを削除するとうまくいくことがある。

GatsbyJSコマンド

※付きは、プロジェクトのルートディレクトリにて実行のこと。

gatsby new
Gatsbyサイトを作成する。オプションを指定しない場合は対話型。Starterを利用する場合は、ディレクトリ名と URLを指定する。例えば hello-worldの Starterを試したければ、「gatsby new hello-world https://github.com/gatsbyjs/gatsby-starter-hello-world」とすればいい。
gatsby develop
開発サーバを起動する。（※）
gatsby build
デプロイ用にコンパイルする。（※）
gatsby serve
「gatsby build」して生成されたサイトを、ローカル開発環境で確認する。（※）
gatsby clean
.cacheディレクトリと publicディレクトリ内を消去する。（※）
gatsby info
バグを報告するときに必要となる環境情報を表示する。OSや CPU、Node.js、npmのバージョン情報、それと、インストール済みのパッケージ情報も確認できる。（※）
