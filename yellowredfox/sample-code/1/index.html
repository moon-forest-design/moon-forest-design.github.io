【React入門】
Hello World!

デモ

コード概要

要点

  • reactを試すために CDNを利用。
    • 開発者用とプロダクション用の CDNが用意されている。開発者用を「開発するときに使用するもの」と解釈してしまったが勘違いだったようだ。開発者用はおそらく、いわゆる「開発者が中身を見たい」ときに使うためのものと考えたほうが近しい。ゆえに圧縮・最適化されていない。CDNを利用する場合に指定するのはプロダクション用。react.production.min.js、react-dom.production.min.jsを使う。
    • また、babel.min.jsも読み込む必要がある。本来は、事前にコンパイルしておけばいいことなので、あまり好ましくはないが仕方がない。
  • Reactは、JavaScriptのライブラリ。仮想DOM（Virtual DOM）という形であれこれ作成できる。JavaScriptで生成した DOMを模倣したデータ構造の差分を、実際の DOMに反映する形。ゆえに、部分的な更新が多いページにおいては処理が高速になるメリットがある。
  • ReactDOM.render()にて、HTMLを差し込むことができる。第一引数に element（HTML文）。第二引数に container（差し込み先）。今回は、空のdiv要素（#root）に対して、p要素を書き加えている。script要素には、type属性で "text/babel"を指定。何かしらの問題がなければ、「Hello React!」が表示される。
  • 第一引数で指定する要素に class属性を使用したらエラーが出た。react（JSX）では className属性を使用する。
  • script要素の内容を外部呼び出し（main.jsあるいはmain.jsx）にしてみたがエラーが出た。クロスオリジンエラー。JSXを外部ファイルで書き出すのであれば、サーバ環境が必要になる。

コード

HTML + React


<!-- reactを使用するために必要なものをCDNで読み込む -->
<script src="https://unpkg.com/react@17/umd/react.development.js"></script>
<script src="https://unpkg.com/react-dom@17/umd/react-dom.development.js"></script>
<script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>

<!-- reactを使って表示するためのコード -->
<div id="root" class="reactRoot"></div>
<script type="text/babel">
  ReactDOM.render(
    <p>Hello World!</p>,
    document.getElementById('root')
  );
</script>

編集後記

あとがき

  • これだけだと Reactにどのようなメリットがあるのか分かりにくいけれども、Reactを理解する上での基礎の基礎となる。
  • トランスパイラ「Babel」は、モダンなJavaScriptコードを古いブラウザでも実行できるよう下位互換バージョンにコード変換するためだけのものと思っていたが、それだけではなかったようだ。Reactにおいては、JSXを実行可能な JavaScriptコードへ変換してくれる役割があるらしい。
