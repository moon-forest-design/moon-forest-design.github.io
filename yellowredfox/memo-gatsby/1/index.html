GatsbyJS概要まとめ

概要

GatsbyJSとは？

  • 高速表示が期待できる静的サイトジェネレータのこと。

GatsbyJSの利点と課題

  • 利点
    • 表示が高速。静的サイトゆえに、一般的なこれまでの CMSのように直接的に DBを使用してサーバ上で HTMLファイルの生成をしない。また、Reactをベースにしているので、ページ内の部分的な更新処理ができるため速い。
    • 動的な表示もできる。基本的に静的なサイトだが、Headless CMSや GraphQLなどを利用することで動的な表示も可能。
    • 習熟度にあわせて導入できる。すでに用意されている Starterやテーマを利用すれば、エンジニアでなくても難しくはない。ゼロから制作する場合は、エンジニアであれば難しくないが、やりたいことの分だけ難易度は増していく。
  • 課題
    • 更新頻度が高いサイトや大規模なサイトには向かない。Next.jsなどを利用するほうがいい。棲み分け。
    • 広く利用されてはいないので、導入に不安がある。情報量、継続性などは現状においてまだまだ懸念される。

GatsbyJS Tips

GatsbyJS関連用語

ページコンポーネント
index.js、about.js、blog.js、404.jsなどの src/pages下に作成するファイル群のこと。例えば、index.jsはトップページ（Home）を単一ページとして表示する。
ビルディングブロックコンポーネント
header.js、footer.js、layout.jsなどの src/components下に作成するファイル群のこと。ページコンポーネント内の一部を構成するために利用される。例えば、header.jsを作成し、各ページコンポーネントから呼び出して、ヘッダーとして表示する。
プラグイン
GatsbyJSで制作した Webサイトに機能を追加するための npmパッケージのこと。必要な機能だけ別個にインストールする。車輪の再発明に陥らずに済む。
gatsby-config.js
Webサイトに関する情報を記述しておく特別な設定ファイル。サイトの metaデータやプラグインの構成、GraphQLなどの設定情報を記述しておくと、GatsbyJSが自動的に読み込んで処理してくれる。
gatsby-browser.js
ブラウザの処理に関する情報を記述しておく特別な設定ファイル。グローバルCSSを設定したい場合や、Gatsby Browser APIsを利用したいときに使用する。
データレイヤー
GatsbyJSでは、データの取得・追加に GraphQLという技術を利用している。各ソースからソースプラグインを介してデータレイヤーにデータを取得し、GraphQLクエリを介して各コンポーネントへ追加する。
GraphiQLインターフェース
http://localhost:8000/___graphql で表示できる。GraphQLクエリを試すことができる。
useStaticQuery
GraphQLクエリをビルディングブロックコンポーネントに追加できるフック。
webhook
運用中の Webサイトが更新されたときに、他の外部サービスへ更新を伝えて反映させる仕組み。
MDX
Markdown + JSXファイル。拡張子は、.mdx。
フロントマター
Frontmatter。MDXなどのファイル内に記述する情報。投稿タイトルや公開日、著者名など、コンテンツに関する情報を記述する。実際のページには直接表示されない仕組み。3つのハイフン（---）の開始セットと終了セットの間に記述する。
File System Route API
src/pagesディレクトリ内のファイルに名前を付けるための特別な構文を定義することで、新しいページを動的に作成するAPI。
クエリ変数
Gatsbyでは、クエリ変数はページクエリ内でのみ使用できる。useStaticQueryフックでは使用できない。
StaticImageコンポーネント
静的に画像を表示したいときの仕組み。gatsby-plugin-imageなどを使う。
GatsbyImageコンポーネント
動的に画像を切り替えたいときの仕組み。

GatsbyJSのおすすめプラグイン

画像プラグイン
画像表示に使用する。最適化機能を有する。gatsby-plugin-imageなど。
ソースプラグイン
各所にあるデータをGraphQLを利用して取得する。gatsby-source-filesystem、gatsby-source-contentfulなど。
MDXトランスフォーマープラグイン
ソースプラグインからコンテンツを取得し、より使いやすいものに変換する。gatsby-plugin-mdxなど。
画像トランスフォーマープラグイン
画像の拡張子（.pngや.jpgなど）で終わるファイルノードを探し、そのファイルのImageSharpノードを作成する。gatsby-transformer-sharpなど。
コメントプラグイン
Markdownに機能を追加するために使用する。gatsby-remark-images、gatsby-remark-prismjs、gatsby-remark-autolink-headers。
MDXプラグイン
MDXを実装する。MDX実装を Reactコンポーネントにマップする。@mdx-js/mdx、@mdx-js/react。
