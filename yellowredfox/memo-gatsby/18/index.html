【GatsbyJS入門】
ページネーションを表示する方法

概要

要点

  1. 状況を整理する。今回のコードは、次のような場合に使用する。状況が違うようなら、パスなどをいじる必要があるかもしれない。
    • マークダウンファイルは、mdx形式にしてある。
    • /src/pages下に blogディレクトリを配置している。
    • 表示上は 1ページ目を、https://sample.com/blog/ 、2ページ目は、https://sample.com/blog/2 としたい。
  2. 「Adding Pagination | Gatsby」を参考にして、gatsby-node.jsを作成する。クエリに slugを生成するために必要。
  3. 「Pagination in GatsbyJS | Nicky blogs」を参考にして、BlogList.jsを作成する。クエリを利用して、動的にブログリストページを作成するために必要。
  4. 「gatsbyJSサイトにページネーションを追加する | Vanilla note」を参考にして、prevと nextのパスを調整する。
  5. /src/pages/blog下に index.jsがあれば、バックアップ後に削除する。

手順

gatsby-node.js


const path = require("path")
const { createFilePath } = require("gatsby-source-filesystem")

exports.createPages = async ({ graphql, reporter }) => {
  const result = await graphql(`
    {
      pageNation: allMdx(
        sort: { fields: [frontmatter___date], order: DESC }
        limit: 2000
      ) {
        edges {
          node {
            slug
          }
        }
      }
    }
  `)
  if (result.errors) {
    reporter.panicOnBuild(`Error while running GraphQL query.`)
    return
  }
  const blogPosts = result.data.pageNation.edges
  const postsPerPage = 10
  const numPages = Math.ceil(blogPosts.length / postsPerPage)
  Array.from({ length: numPages }).forEach((_, i) => {
    createPage({
      path: i === 0 ? `/blog/` : `/blog/${i + 1}`,
      component: path.resolve("./src/templates/blogList.js"),
      context: {
        limit: postsPerPage,
        skip: i * postsPerPage,
        numPages,
        currentPage: i + 1,
      },
    })
  })
}

exports.onCreateNode = ({ node, actions, getNode }) => {
  const { createNodeField } = actions
  if (node.internal.type === `Mdx`) {
    const value = createFilePath({ node, getNode })
    createNodeField({
      node,
      name: `slug`,
      value,
    })
  }
}

src/templates/BlogList.js


import React from "react"
import { Link, graphql } from "gatsby"
import Layout from "../components/layout"

const BlogList = ({ pageContext, data }) => {
  const posts = data.allMdx.edges
  const { currentPage, numPages } = pageContext
  const isFirst = currentPage === 1
  const isLast = currentPage === numPages
  const prevPage = currentPage - 1 === 1 ? "/" : (currentPage - 1).toString()
  const nextPage = (currentPage + 1).toString()
  const prevPath = prevPage === "/" ? "/blog" : "/blog/"
  const nextPath = "/blog/"
  return (
    <Layout>
      {posts.map(({ node }) => {
        const slug = node.slug;
        const { title } = node.frontmatter;
        return (
          <div key={slug}>
            <Link to={`/blog/${slug}`}>
              {title || slug}
            </Link>
          </div>
        );
      })}
      {!isFirst && (
        <Link to={`${prevPath}${prevPage}`} rel="prev">
          ←
        </Link>
      )}
      {Array.from({ length: numPages }, (_, i) => (
        <Link key={`pagination-number${i + 1}`} to={`/blog/${i === 0 ? "" : i + 1}`}>
          {i + 1}
        </Link>
      ))}
      {!isLast && (
        <Link to={`${nextPath}${nextPage}`} rel="next">
          →
        </Link>
      )}
    </Layout>
  )
}
export const blogListQuery = graphql`
  query blogListQuery($skip: Int!, $limit: Int!) {
    allMdx(
      sort: { fields: [frontmatter___date], order: DESC }
      limit: $limit
      skip: $skip
    ) {
      edges {
        node {
          slug
          frontmatter {
            title
          }
        }
      }
    }
  }
`

export default BlogList;

編集後記

あとがき

  • gatsby-node.jsで少々苦戦した。パスの指定が間違っていたようだ。慣れないうちは、実際のディレクトリのパス名を指定すればいいのか、ブラウザで表示するときのパス名を指定するのか、あるいは、/（スラッシュ）の有無などもややこしい。
  • 動的に blogページや tagsページを作るために gatsby-node.jsを使用している場合は、整理する手間が増えて大変かもしれない。これもまたややこしい。
