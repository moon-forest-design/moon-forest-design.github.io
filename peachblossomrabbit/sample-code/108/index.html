React入門
「関数によるコンポーネント化」

コード概要

要点

  • 前回の React入門 Hello World!の続き。Reactの「コンポーネント」と呼ばれる書き方。
  • 例えば、ReactDOM.render()の第一引数に直接 HTML要素を書くのではなく、関数のような形で定義する。機能的に分割できるひとまとまりがコンポーネント（部品）。このように分割しておくことで再利用しやすくなる。
  • コンポーネント名は、一文字目を大文字にしなければいけない。例えば、サンプルコードでよく見かける <App />も一文字目が「A」になっている。試しに <textView />にしたらダメで <TextView />なら OKだった。これは JSXのルールらしい。
  • React.StrictModeは、おそらく JavaScriptの "use strict";みたいなものだろう。

コード

HTML + React


<div id="root" class="reactRoot"></div>
<script type="text/babel">
  function App() {
    return <p>Reactテスト：コンポーネントを使う（関数で定義）</p>;
  }
  ReactDOM.render(
    <React.StrictMode>
      <App />
    </React.StrictMode>,
    document.getElementById('root')
  );
</script>

編集後記

あとがき

  • 関数名の一文字目を大文字にするルールは忘れやすい。はじめのうちはとくに忘れやすい。うっかりしてなぜ動かないのか気づかずに無駄に時間をすごすことになりがち。気をつけよう。
  • 関数で書く方法以外にも ES6の classで定義する方法もある。他にもいくつか書き方があるようだが、廃止になったものもある。古いコードを見たときに、はじめのうちは混乱するかもしれない。
