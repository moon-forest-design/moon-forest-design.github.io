React入門
「buttonイベント基礎」

コード概要

要点

  • Reactのイベント処理は、on系の属性を使用する。（例：onClick属性、onChange属性など）
  • onClick={...}のように書き、中には関数を指定する。関数内には、イベントハンドラを書く。イベントハンドラとは、ユーザの操作により発生した要件に対して処理すべき命令群）今回のサンプルでは、ボタンがクリックされたら「クリックされた」とコンソール表示されるようにアロー関数で書いている。
  • 以下は、同じ仕様のボタンだが、onClick属性内に処理を書かずに、コールバック関数として外に書いた場合のサンプル。

コード

HTML + React


<!-- onClick属性内に処理を書いた場合のコード -->
<div id="root" class="reactRoot"></div>
<script type="text/babel">
  const ClickBtn = () => {
    return (
      <div>
        <button onClick={() => console.log('clicked 1')}>Click me!</button>
      </div>
    )
  }
  ReactDOM.render(
    <ClickBtn />,
    document.getElementById('root')
  );
</script>

<!-- onClick属性外に処理を書いた場合のコード -->
<div id="root2" class="reactRoot"></div>
<script type="text/babel">
  const ClickBtn2 = () => {
    // コールバック関数
    const handleBtn= () => {
      // イベントハンドラを書く
      console.log('clicked 2')
    }
    return (
      <div>
        <button onClick={handleBtn}>Click me!</button>
      </div>
    )
  }
  ReactDOM.render(
    <ClickBtn2 />,
    document.getElementById('root2')
  );
</script>

編集後記

あとがき

  • Reactでは、イベントを表す propsには on[Event] という名前、イベントを処理するメソッドには handle[Event]という名前を付けるのが慣習となっている。
  • より実践的なコードは、どのような場面で使用する技術なのかイメージしやすく分かりやすいが、いかんせん内容が複雑になりがちだ。今回のサンプルのように、シンプルなコードも見て慣れていくと理解しやすい。
